# KRAMANIK — Deployment Guide
# ============================================================
# Two options: Local Docker OR Cloud (Railway + Netlify)
# ============================================================


## OPTION A: LOCAL DOCKER (run everything on your machine)
## Easiest — good for demos and testing
## ─────────────────────────────────────────────────────────

### Prerequisites
- Docker Desktop installed (https://www.docker.com/products/docker-desktop)
- Git

### Steps

1. Clone the repo (or unzip the project folder)

2. Copy the env file:
   cp .env.example .env

3. Edit .env — change the passwords:
   DB_PASSWORD=your_strong_password
   JWT_SECRET=your_64_char_random_string

4. Start everything:
   docker compose up --build

5. Wait ~60 seconds for MySQL to initialise and Spring Boot to start.

6. Open browser:
   Frontend: http://localhost
   Backend:  http://localhost:8080/actuator/health

7. Login with the demo account:
   Email:    admin@kramanik.in
   Password: Admin@123

### Stop
   docker compose down

### Reset database
   docker compose down -v   (removes the DB volume — fresh start)


## OPTION B: CLOUD DEPLOYMENT (Railway + Netlify)
## Production-ready — gives you a real public URL
## ─────────────────────────────────────────────────────────

### STEP 1: Push to GitHub

   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/YOUR_USERNAME/kramanik.git
   git push -u origin main


### STEP 2: Deploy backend on Railway

   a. Go to https://railway.app and sign up (free tier available)

   b. New Project → Deploy from GitHub repo → select your kramanik repo

   c. Railway auto-detects railway.json and uses the Dockerfile

   d. Add a MySQL plugin:
      Railway dashboard → + New → Database → MySQL
      Railway auto-sets MYSQL_URL in your service.

   e. Set these environment variables in Railway dashboard:
      (Project → your service → Variables tab)

      SPRING_DATASOURCE_URL     = (Railway provides this from MySQL plugin)
      SPRING_DATASOURCE_USERNAME = (from MySQL plugin)
      SPRING_DATASOURCE_PASSWORD = (from MySQL plugin)
      KRAMANIK_JWT_SECRET        = (generate: openssl rand -hex 64)
      KRAMANIK_CORS_ALLOWED_ORIGINS = https://YOUR_NETLIFY_APP.netlify.app

   f. Wait for deploy. Railway gives you a URL like:
      https://kramanik-backend-production.up.railway.app

   g. Run schema.sql on the Railway MySQL:
      Railway dashboard → MySQL plugin → Connect → run schema.sql contents


### STEP 3: Update frontend API URL

   Open frontend/js/config.js and set your Railway URL:

      const PRODUCTION_API_URL = 'https://YOUR_RAILWAY_URL.railway.app/api';

   Commit and push this change.


### STEP 4: Deploy frontend on Netlify

   a. Go to https://netlify.com and sign up

   b. New site → Import from Git → select your kramanik repo

   c. Build settings (netlify.toml handles this automatically):
      Publish directory: frontend
      Build command: (leave empty)

   d. Deploy. Netlify gives you a URL like:
      https://kramanik-abc123.netlify.app

   e. Go back to Railway and update KRAMANIK_CORS_ALLOWED_ORIGINS
      to your real Netlify URL, then redeploy backend.

   f. (Optional) Set a custom domain in Netlify:
      Site settings → Domain management → Add custom domain
      e.g. kramanik.in


### STEP 5: Verify

   Open your Netlify URL → login with admin@kramanik.in / Admin@123
   Dashboard should load with live data from Railway backend.


## ENVIRONMENT VARIABLES REFERENCE
## ─────────────────────────────────────────────────────────

| Variable                      | Where to set     | Example value                        |
|-------------------------------|------------------|--------------------------------------|
| SPRING_DATASOURCE_URL         | Railway          | jdbc:mysql://mysql.railway.internal/kramanik_db |
| SPRING_DATASOURCE_USERNAME    | Railway          | root                                 |
| SPRING_DATASOURCE_PASSWORD    | Railway          | auto-generated by Railway            |
| KRAMANIK_JWT_SECRET           | Railway          | (64 char random hex)                 |
| KRAMANIK_CORS_ALLOWED_ORIGINS | Railway          | https://kramanik.netlify.app         |
| PRODUCTION_API_URL            | config.js (code) | https://xyz.railway.app/api          |


## TROUBLESHOOTING
## ─────────────────────────────────────────────────────────

Problem: Backend crashes on startup
Fix:     Check Railway logs. Usually it's DB connection. Verify SPRING_DATASOURCE_* vars.

Problem: CORS error in browser console
Fix:     Update KRAMANIK_CORS_ALLOWED_ORIGINS in Railway to match your exact Netlify URL.

Problem: Login says "Invalid credentials" on production
Fix:     schema.sql seed data (demo@kramanik.in / Admin@123) may not have been run.
         Run it manually in Railway MySQL console.

Problem: Docker compose fails
Fix:     Run: docker compose down -v && docker compose up --build
         This resets the DB volume and rebuilds images.
